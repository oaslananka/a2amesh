# Professionalization plan

## Objective

Move A2A Mesh from an incubating-like solo-maintainer public package toward Professional OSS / Mature OSS without overclaiming Gold or foundation-grade maturity.

## Current safe scope

- Documentation-only professionalization.
- Metadata-only maturity evidence.
- README information architecture.
- Process and policy documentation.

## Not in automatic scope

- Public API changes.
- CLI behavior changes.
- Package manager changes.
- Publish workflow or registry credential changes.
- Required human review enforcement before an independent reviewer exists.
- Gold/foundation-grade claims.

## Milestones

### M0: Repository health baseline

Keep community standards files, CI/docs/security/CodeQL/Scorecard required checks, branch protection, and OpenSSF evidence current.

### M1: Release integrity

Resolve Release Please flow, confirm npm trusted publishing/OIDC, and publish release notes, checksums, SBOM/provenance evidence where applicable.

### M2: Contributor process

Add curated good-first/help-wanted issues, recruit an independent reviewer, and enable required PR review only after independent review is sustainable.

### M3: Silver readiness

Confirm private vulnerability reporting, release integrity evidence, and OpenSSF Best Practices BadgeApp evidence.

### M4: Gold feasibility

Gold requires multiple maintainers, independent human review, contributor diversity, governance succession, and measured response/release cadence.
