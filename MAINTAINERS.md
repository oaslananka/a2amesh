# Maintainers

The A2A Mesh repository is maintained by:

| Area                                        | Maintainer  |
| ------------------------------------------- | ----------- |
| Repository, releases, and security response | @oaslananka |

Security reports must use [GitHub private vulnerability reporting](https://github.com/oaslananka/a2amesh/security/advisories/new).

## Responsibilities

Maintainers are responsible for:

- reviewing and merging pull requests;
- enforcing branch-protection, required-check, and release policies;
- triaging issues and security reports;
- maintaining release notes and versioning discipline;
- dispatching release/publish workflows only after required checks pass;
- rotating credentials and revoking compromised access;
- updating the security policy, roadmap, and OpenSSF evidence when project practice changes.

## Continuity status

The project currently has one repository admin listed above. The project is actively tracking the need for an independent maintainer in [#125](https://github.com/oaslananka/a2amesh/issues/125). Once a second trusted maintainer is available, this file should list their role and minimum permissions so that issue triage, PR review, release dispatch, and vulnerability response can continue if any one maintainer becomes unavailable.
